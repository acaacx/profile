Speaking Points
Slide 1 of 5

1. Deterministic vs. Agentic Testing

LLM agents are non-deterministic. Static assertions fail to validate multi-step agent behaviors. Our pipeline executes the agent codebase inside isolated runtimes to assess runtime pathing and tool calls before integration.

Core Concept Non-deterministic pathing: Agents can resolve the same goal via different tool combinations.

2. Auditing Tool Trajectories

If an agent resolves a task (e.g., patching a file), standard tests only verify the final file state. They miss pipeline-breaking behaviors: infinite loops, security policy violations (e.g., executing unvetted bash commands), or runaway API costs.

Solution We monitor the exact sequence and parameters of tool calls (**Execution Trajectory**).

3. Golden Dataset Sandbox

We maintain a version-controlled benchmark of 10 to 100 historical edge cases. The pipeline provisions ephemeral container environments with mocked datastores and fixed API budgets, running the agent suite in parallel.

Goal Standardized, repeatable regression tests run with zero access to production systems.

4. Scoring Engine Metrics

The evaluator outputs two primary signals:

  • Trajectory Overlap: Graph similarity of the agent's actions compared to reference trajectories.
  • Accuracy / ROUGE-L: Lexical and semantic overlap of the agent's final output against reference outputs.

Gate check A weighted combination of both metrics determines the final quality score.

5. Automated Release Decision

Conditional pipeline routing:

  • Score < Threshold: Evaluator container exits with Code 1. Cloud Build halts the release and flags the GitHub PR check as failed.
  • Score >= Threshold: Evaluator container exits with Code 0. Cloud Build packages the agent and rolls it out to Production Cloud Run.

Result Empirically gates agent releases against regression and safety violations.
